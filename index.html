A Adresă IP

Numărul care identifică un dispozitiv în rețea.

A Algoritm

Pași clari, în ordine, pentru a rezolva o problemă.

A Alocare

Rezervarea de memorie sau resurse pentru o variabilă sau obiect.

A Amprenta digitală (online)

Urmele pe care le lași când folosești internetul.

A Animație (Scratch)

Schimbarea costumelor pentru a crea mișcare.

A Argument (funcție)

Valoare trimisă unei funcții ca să lucreze cu ea.

B

B Backpack (Scratch)

Loc pentru a salva blocuri și costume între proiecte.

B Bază (numerație)

Sistemul în care scriem numerele: baza 10, 2, 16.

B Big O

Felul în care estimăm cât de repede crește timpul unui algoritm.

B Bit

Cea mai mică unitate de informație: 0 sau 1.

B Bloc (Scratch)

Cărămidă de cod pe care o tragi în scenariu.

B Bluetooth

Conectare fără fir pe distanțe scurte între dispozitive.

B Boolean

Tip cu două valori: adevărat sau fals.

B Bug

Greșeală în program care dă un comportament ciudat.

C

C Cache

Loc rapid unde se păstrează temporar date folosite des.

C Caracter

Un singur simbol: literă, cifră sau semn.

C Clasă (OOP)

Șablon din care facem obiecte cu aceleași proprietăți.

C Client

Programul sau calculatorul care cere informații de la server.

C Clone (Scratch)

Copii temporari ai unui personaj, create prin cod.

C Cod

Instrucțiuni pe care le înțelege calculatorul.

C Compilator

Transformă codul în limbaj de mașină înainte de rulare.

C Condiție

Întrebare cu răspuns adevărat sau fals folosită la decizii.

D

D Date

Informații pe care le stocăm și le procesăm.

D Debug

Căutarea și repararea erorilor din cod.

D Decizie (if)

Execută cod doar dacă propoziția e adevărată.

D Decompoziție

Împărțim problema mare în bucăți mai mici.

D Definiție (funcție)

Locul unde scriem ce face funcția.

D Director (folder)

Un „dosar” în care păstrăm fișiere.

D Dispozitiv

Orice aparat: PC, tabletă, telefon, micro:bit.

D Divizor

Număr care intră perfect în altul, fără rest.

E

E Edge (margine)

Limita scenei în Scratch; locul unde se oprește sprite-ul.

E Ecran

Suprafața pe care se afișează aplicația sau jocul.

E Editor

Locul unde scrii sau aranjezi codul.

E Element (listă)

O piesă dintr-o listă sau tablou.

E Eveniment

Ceva ce se întâmplă și pornește codul: clic, tastă, mesaj.

E Executare

Momentul când programul rulează.

E Expresie

Combină valori și operatori pentru a obține un rezultat.

E Extensie (fișier)

Finalul numelui de fișier care arată tipul: .sb3, .py, .hex.

F

F Fail-safe

Scrierea codului astfel încât să nu se strice tot la o eroare.

F Flag (stare)

Variabilă care arată o stare: pornit/oprit, găsit/negăsit.

F Fișier

Locul unde păstrăm datele pe disc.

F Flowchart

Diagrama pașilor unui algoritm cu forme și săgeți.

F Flowgorithm

Program pentru a face și rula flowchart-uri.

F For (buclă)

Repetă pașii de un număr cunoscut de ori.

F Funcție

Bucată de cod care primește valori și întoarce un rezultat.

F Firmware

Program mic din interiorul unui dispozitiv (ex.: micro:bit).

G

G Git

Sistem pentru a salva versiuni ale codului și a colabora.

G GPU

Procesor pentru grafică; accelerează imagini și jocuri.

G GPS

Sistem care spune poziția pe hartă a unui dispozitiv.

G Gadget

Dispozitiv mic cu o funcție utilă.

G Game loop

Bucla principală a unui joc: actualizează și desenează.

G Gateway

Punct care leagă o rețea de alta.

G GIF

Format de imagine care poate avea animație.

G GPIO (micro:bit)

Pin prin care micro:bit citește sau controlează semnale.

H

H HTTP

Regulile după care se trimit paginile web.

H HTML

Limbajul cu care se construiesc paginile web.

H Hardware

Partea fizică a calculatorului sau dispozitivului.

H Heap

Zonă de memorie pentru obiecte create în timpul rulării.

H Header

Partea de sus a unei pagini sau a unui pachet de date.

H HID

Dispozitiv de intrare: tastatură, mouse, controller.

H Hz (Hertzi)

I

I Identificator

Numele unei variabile, funcții sau clase.

I IF (instrucțiune)

I Imagine vectorială

I Import

Aduce cod din alte fișiere sau biblioteci.

I Indentație

Spații la începutul liniilor care arată structura codului.

I Input

Date care intră în program (taste, senzori, fișiere).

I Internet

Rețea uriașă care leagă calculatoare din toată lumea.

J

Limbaj de programare folosit în aplicații și Android.

J JavaScript

Limbaj care face paginile web interactive.

J Joystick

Control pentru jocuri, cu butoane și pârghie.

J JSON

Format simplu pentru a schimba date (cheie: valoare).

J Jurnal (log)

Lista cu mesaje despre ce a făcut programul.

J JIT

Traducere a codului în timp ce rulează, pentru viteză.

J Job (task)

O sarcină pe care o execută programul sau sistemul.
